# GeorgeLoops Vision

GeorgeLoops should be the catalog of recurring AI loops that are useful enough
to run again: scheduled scans, maintenance rituals, review passes, and bounded
agent workflows.

## Product Thesis

Good loops are boring. They watch a narrow surface, produce a bounded report,
and make the next human decision easier. The value is not autonomy for its own
sake; the value is turning repeated friction into named, reviewable operating
systems.

## Goals

- Start loops in report-only mode.
- Define cadence, inputs, workflow, outputs, guardrails, and verifier for each
  loop.
- Keep public-facing loop docs free of secrets and machine-specific paths.
- Promote loops only after repeated useful runs.

## Non-Goals

- Do not store content drafts, social scripts, or private source data here.
- Do not add destructive actions without an explicit approval gate.
- Do not treat vague persistence as a real loop.
